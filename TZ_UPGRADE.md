# Обновленное ТЗ и рекомендации

## 1) Как улучшить исходное ТЗ, чтобы сайт выглядел современно

### Что добавить в требования
- **Четкую цель сайта**: например, «получение заявок от госзаказчиков».
- **Основной CTA на каждом экране**: «Оставить заявку» + «Скачать презентацию».
- **Единый визуальный стиль**: темная/светлая тема, палитра, типографика, иконки.
- **Адаптивность**: обязательные версии для мобильного (360px+), планшета (768px+), десктопа (1280px+).
- **Производительность**: WebP/AVIF для изображений, lazy-load, сжатие ассетов.
- **Доверие и юридический блок**: ИНН/ОГРН, реквизиты, политика обработки персональных данных.
- **SEO-база**: корректные title/description, H1-H2 структура, OpenGraph, schema.org (Organization).
- **Аналитика**: события на CTA-кнопках (Яндекс.Метрика, GA4 при необходимости).

### Что добавить в контент
- Для каждого кейса: 1 фото + задача + решение + результат + цифры.
- Для блока «Нам доверяют»: минимум 8–12 логотипов в SVG/PNG на прозрачном фоне.
- Для услуг: короткие выгоды, а не только перечисление форматов.
- Для «Процесс работы»: сроки этапов (например, «подготовка КП за 2–3 дня»).

## 2) Как передавать фотографии и материалы

Рекомендуемый пакет:
1. Папка `assets/` c подпапками:
   - `logos/` — логотипы заказчиков (желательно SVG или PNG прозрачный фон)
   - `cases/` — фото проектов (минимум 1800px по ширине)
   - `team/` — фото команды (если будут)
2. Файл `content.csv` или `content.xlsx`:
   - название кейса
   - краткое описание
   - год
   - город
   - ссылка на подробный кейс
3. Логотип компании в двух версиях: темная/светлая.
4. Презентация в PDF (финальная версия для кнопки «Скачать презентацию»).

Технические требования к медиа:
- Форматы: `.svg`, `.png`, `.jpg`, `.webp`
- Размер одного файла до 2–4 МБ (до оптимизации)
- Названия файлов латиницей: `case-forum-2025.jpg`

## 3) Что сделано в коде на текущем шаге

Сверстана стартовая версия лендинга по вашему ТЗ:
- Главный экран
- «Нам доверяют»
- «Цифры и опыт»
- «Услуги»
- «Кейсы» (6 карточек + кнопка «Подробнее»)
- «Почему мы»
- «Система реализации проектов»
- «Как мы работаем»
- CTA-блок
- Контакты + форма заявки + кнопка презентации
